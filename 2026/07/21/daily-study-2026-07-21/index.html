🔧 嵌入式知识：SPI 事务、片选、模式与板级电气

工程问题：逻辑分析仪能解码，Flash 为什么仍会偶发读错

实验平台是 STM32F407 Discovery，外接 Winbond W25Q128JV：SPI2 使用 PB13/SCK、PB14/MISO、PB15/MOSI，PB12 作为软件片选，3.3 V 供电并共地。本文只讨论单线全双工 SPI，不把 Quad SPI、DDR、三线半双工或菊链结论直接套进来。W25Q128JV 的指令、片选边界、busy 位和交流时序以 Winbond 官方数据手册为准；STM32F407 的 SPI 寄存器和清错顺序以 ST RM0090 第 28 章为准。

SPI 没有统一“数据帧”。控制器只负责在时钟边沿移位，目标器件把 CS 有效区间解释为一笔事务。W25Q 的普通读事务是 CS↓ -> 0x03 -> 24位地址 -> 数据 -> CS↑；若驱动把命令和地址拆成两次、并在中间抬高 CS，控制器两次都可能报告成功，Flash 却看到两笔残缺事务。

模式、位序与时序裕量是一份合同

CPOL 决定空闲电平，CPHA 决定第一个有效边沿用于采样还是改变。W25Q 支持模式 0 与 3；本文选模式 0，即空闲低、第一边沿采样。STM32 的 CPOL=0, CPHA=0 对应这一选择。CMSIS-Driver 也把模式、位序、数据位宽和软件/硬件片选列为独立控制项，说明“SPI 已初始化”远不等于合同完整；接口语义见 Arm CMSIS-Driver SPI。

全双工意味着读取 ID 时仍要发送占位字节。8 位访问 DR 可避免在 8 位模式下产生额外帧。释放 CS 前等待 BSY 清零，而不是只等 TXE；TXE 仅说明发送缓冲可再写，最后一位可能仍在线上。所有失败路径都释放 CS，否则下一次调用会继承半笔事务。

事务层：写使能、页编程与业务完成

页编程不能跨越 256 字节页边界；写使能锁存器要先由独立 0x06 事务设置；发送完 0x02+地址+数据 只表示串行输入完成，内部编程期间状态寄存器 BUSY 仍为 1。下面函数演示边界检查和轮询。生产代码应把循环换成有时基的毫秒超时，并决定掉电恢复策略。

DMA、并发与片选所有权

DMA 完成中断也不自动等于总线空闲：内存数据搬完后 SPI 可能仍 BSY。事务对象必须持有 CS、SPI 配置和收发缓冲的所有权，直到最后一位发出并释放 CS。若两个 RTOS 任务分别“锁 SPI”和“操作 CS”，它们仍可能交错；正确锁粒度是一整笔器件事务。CMSIS 的非阻塞 Transfer 同样要求完成或取消前缓冲区保持有效，不能传栈上临时数组后立即返回。

多器件共享 MISO 时，未选中目标必须高阻。上电阶段 MCU 引脚尚未配置，外接上拉保证各 CS 无效；若某器件在复位期间驱动 MISO，应核对其复位/供电时序。软件片选便于跨多缓冲保持，但高优先级中断可延长 CS 低电平；只要数据手册没有最大事务间隔通常可接受，要求连续时钟的器件则需硬件 NSS 或 DMA。

可复现实验与通过标准

准备：STM32F407 Discovery、W25Q128JV 模块、100 nF 去耦、逻辑分析仪；写实验另需确认测试扇区可擦除。步骤一运行 flash_jedec_id 一千次，记录三字节 ID。步骤二把 SPI 分频依次改为 256、32、4，捕获 CS/SCK/MOSI/MISO。步骤三读取固定地址 256 次并计算 CRC。步骤四在空白页写入 00 FF AA 55 模式，等待 BUSY 清零后回读。

通过标准：ID 每次一致且厂商字节为 Winbond 定义值；一笔 ID 命令只有一次 CS 低区间；模式 0 的采样边沿与数据稳定区一致；回读 CRC 零差异；页编程不跨页且状态 busy 在超时内清除。示波器测得接收端无越过绝对最大额定的过冲，建立/保持裕量满足手册。

真实故障注入与完整定位链

注入：把 JEDEC ID 分成两笔事务，在发送 0x9F 后立即抬高 CS，再次拉低后读三个占位字节。现象是驱动每字节都成功、收到值却常为 FF 或无意义数据。假设一是模式错误，假设二是器件未供电，假设三是事务边界错误。证据是 SCK 与数据边沿满足模式 0，供电为 3.3 V，而逻辑分析仪清楚显示命令后出现 CS 高脉冲；对照 Winbond 命令图，输出 ID 必须属于同一 CS 有效区间。

定位到上层把“发送命令”和“读取响应”分别调用了会自动控制 CS 的函数。修复是建立 flash_jedec_id 事务接口，由它一次取得总线锁、拉低 CS、完成四个字节、等待 BSY、释放 CS。回归重复不同分频与一千次读取，ID 零错误；再注入 SPI 超时，确认失败路径也释放 CS，下一笔事务可恢复。完整链条是：错值现象 → 三个假设 → 电气/模式/CS 波形证据 → 定位错误抽象边界 → 合并事务修复 → 正常与超时双路径回归。

第二类常见故障是 OVR：只发送而不读取 DR，使 RXNE 堆积。现象可能是后续接收停滞；证据是 SR.OVR 置位。按 RM0090 要求读取 DR 再读 SR 清除，并修复驱动为每发一帧都消费接收数据。不能只在异常后清标志而保留单向发送模型。

超时后的恢复不是简单重发

SPI 没有总线级确认，主机看到的超时可能发生在命令前、地址中间、数据已经全部送达但软件尚未观察完成等不同阶段。对只读命令，释放片选、清控制器错误并重新执行通常是幂等的；对页编程，盲目重发可能重复磨损或覆盖不同数据。恢复函数必须知道指令语义，而不能由字节驱动统一“重试三次”。

控制器侧恢复顺序应有限且可证明：禁止新的 DMA 请求；等待或强制停止当前数据路径；按参考手册消费 DR/SR 清 OVR；释放所有软件片选；必要时关闭再开启 SPI；恢复固定 mode、位宽和分频；最后通过无副作用命令读取 JEDEC ID 或状态寄存器确认总线活着。若 BSY 永不清除，要考虑外设复位，但外设复位会丢配置，也可能影响共享 I2S 功能。

目标器件侧则按状态机恢复。页编程命令的串行部分已结束后，即使 MCU 超时，Flash 仍可能在内部 busy；应先查询状态，等待完成，再回读目标区域判断是否提交。若掉电发生在编程中，数据手册不保证目标页内容，系统必须依靠上层双槽、版本和 CRC 决定哪份记录有效。SPI 驱动只能搬运字节，不能替代掉电原子协议。

总线错误还应记录阶段：命令、地址、数据索引、SR、DMA 剩余计数、CS 状态和目标状态。只记录“SPI error”无法判断是否允许重试。错误日志也不能在持有 SPI 锁时写入同一片 Flash，否则会递归争锁或破坏原事务。

板级多目标与安全边界

多个目标共享 SCK/MOSI/MISO 时，每个 CS 都应有上电默认无效电平。分别只选中一个器件，测量其他目标的 MISO 是否真正高阻；两个推挽输出争用时，数字解码可能偶尔仍正确，但会产生大电流和中间电平。用串联电阻可限制电流，却不能把违反片选高阻合同变成正确设计。

若不同目标要求不同 mode 或最高频率，事务开始必须在 SPI 禁用且总线空闲时重配，不能在前一个 BSY 尚未清除时写 CR1。配置也属于锁保护状态。对带 DMA 的 RTOS 系统，任务取消时要等待取消完成事件，随后才能复用缓冲和释放锁；强行删除持锁任务会留下 CS 有效和 DMA 继续读内存的组合故障。

外部接口还涉及安全：来自不可信目标的长度和状态不能直接决定 DMA 越界，Flash 读回的元数据必须做范围与完整性验证。写保护引脚、状态寄存器保护位和软件权限应共同使用。通信成功只证明收到某些位，不证明内容可信或新鲜。

把波形转成可回归的数据

一次漂亮截图不足以说明量产裕量。测试应导出逻辑分析仪数据，自动检查每笔事务的 CS 低电平次数、字节数、命令、地址和间隔；对不同温度、电压、线长和分频统计错误率。示波器则保存目标端 SCK 的高低脉宽、过冲、振铃稳定时间以及数据相对采样边沿的建立保持。

通过门槛应来自最坏条件：若目标手册要求最小 CS 高时间，自动脚本取全部事务最小值而不是平均值；若系统允许中断拉长字节间隔，验证目标是否规定最大间隔。任何一次超时后都追加一笔 ID 读取和已知数据 CRC，证明恢复路径没有把错误带到下一位使用者。

练习题与验收标准

  1. 为什么 TXE=1 不能立即抬高 CS？合格答案必须指出发送缓冲空不等于移位器空，并使用 BSY 判断最后一位结束。
  2. W25Q 页末剩 8 字节却请求写 16 字节，接口应怎样处理？标准是拒绝或拆为两页事务，不能让器件地址在页内回卷后静默覆盖。
  3. 低速成功、高速失败应收集什么证据？至少包括目标端 SCK/MOSI 波形、模式边沿、CS 建立保持、供电与回流；只建议继续降频不合格。
  4. 设计共享总线锁的边界。答案必须覆盖配置、CS、所有数据段、BSY 和错误恢复，而不是只锁单次字节传输。
  5. 实操验收：提交一张包含完整 0x9F 事务的四通道波形和一千次 ID 统计；CS 中断、任一 ID 不一致或超时后不能恢复均判失败。

搭配与语境

  • establish · establish a system / establish a link

    • Air pollution is a major cause of preventable illness.
  • principle · a basic principle / in principle
    • Equal access is a basic principle of public education.
  • research · conduct research / research findings
    • Further research is needed before the policy is expanded.
