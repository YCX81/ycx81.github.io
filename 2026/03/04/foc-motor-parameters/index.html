FOC 电机参数详解：从获取到实际应用

发表于 2026-03-04 分类于 嵌入式开发 本文字数： 2.6k 阅读时长 ≈ 9 分钟

本文以 pmsm_2312s_init() 中 2312S 电机参数为例，逐一解释每个参数的含义、实际获取方法，以及在 FOC 控制中的用途。

一、参数总览

电机电气参数

代码 值 含义
pn = 7 7 极对数，电角度 = 机械角度 × 7
Rs = 0.203 Ω 定子相电阻，影响铜耗和电流环
Ld = 108.8 μH d 轴电感（直轴，磁场方向）
Lq = 112.4 μH q 轴电感（交轴，出力方向）
Ls = 110.6 μH 平均电感 (Ld+Lq)/2，用于简化模型
Ldif = 3.64 μH 电感差 (Lq-Ld)/2，凸极性，高频注入用
flux = 6.488 mWb 永磁磁链，反电动势常数的核心

电机机械参数

代码 值 含义
B = 1.88e-4 N·m·s/rad 粘滞摩擦系数，速度相关阻力
Js = 9.09e-5 kg·m² 转动惯量（约 90 g·cm²），影响加减速响应

控制/传动参数

代码 值 含义
Gr = 1.0 — 减速比，1.0 = 直驱无减速
ibw = 1000.0 Hz 电流环期望带宽，用于自动整定 PI 参数
delta = 4.0 — 阻尼比系数，电流环相位裕度/阻尼参数

预计算常量（避免实时除法）

代码 计算 含义
div_pn 1/7 极对数倒数，电角速度 → 机械角速度
pnd_2pi 7/2π 机械圈数 → 电圈数换算
div_Gr 1.0 减速比倒数
Kt 1.5×7×0.006488 ≈ 0.0681 N·m/A 转矩常数，iq 电流 → 电磁转矩
div_Kt ≈ 14.68 转矩常数倒数，转矩 → 电流换算

运动控制参数

代码 值 含义
wm_acc = 200 rad/s² 加速度，速度上升斜率限制
wm_dec = 200 rad/s² 减速度，速度下降斜率限制

角度偏移（标定结果）

代码 值 (rad) 含义
e_off = 2.344 ≈ 134.2° 编码器电角度偏移
r_off = 2.130 ≈ 122.1° 转子机械角度偏移
m_off = 0.0 0° 附加偏移，预留校正量

这三个偏移角通过编码器标定获得，确保 Park 变换的角度与实际转子磁极位置对齐，是 FOC 正确运行的前提。

二、参数的实际获取方法

1. 极对数 pn

最简单的参数，三种方法：

  • 查数据手册：电机厂商直接提供
  • 手动法：缓慢转动转子一圈，用示波器数反电动势的波形周期数
  • 驱动法：给定开环电压缓慢旋转，观察电角度转了几圈 ÷ 机械转一圈 = 极对数

2. 相电阻 Rs

方法一：万用表直测

  • 测量任意两相线间电阻，除以 2 = 单相电阻
  • 精度一般，受接触电阻和温度影响

方法二：直流注入法（推荐，可软件自动化）

  • 给 d 轴注入恒定小电流 Id，q 轴电流为 0
  • 电机静止不动时：Vd = Id × Rs
  • 直接计算：Rs = Vd / Id

3. 电感 Ld、Lq、Ls

方法一：LCR 表直测

  • 锁定转子在 d 轴和 q 轴位置分别测量
  • 缺点：测试频率与实际运行频率不同，结果有偏差

方法二：高频注入法（推荐）

  • 在 d 轴或 q 轴注入高频小信号电压（如 1kHz 方波）
  • 根据电流响应的上升斜率计算：

方法三：阶跃响应法

  • 注入 d 轴阶跃电压，记录电流上升曲线
  • 拟合一阶系统：τ = L/R，已知 Rs 即可算出 L
  • 这是最常用的工程做法

Ls = (Ld + Lq) / 2，Ldif = (Lq - Ld) / 2，辨识出 Ld/Lq 后直接计算。

4. 磁链 flux

方法一：反电动势法（最常用）

  • 用另一台电机拖动待测电机匀速旋转
  • 示波器测量线反电动势峰值 Vll_peak
  • 计算：flux = Vll_peak / (√3 × ω_e)

方法二：软件辨识（开环拖动法）

  • 开环 V/F 拖动电机到稳定转速
  • 空载时 Iq ≈ 0，所以：flux ≈ Vq / ω_e
  • 正是 drag_vf 模式能做的事

方法三：闭环静态法

  • d 轴通入已知电流 Id，q 轴电流为 0，电机静止
  • 逐步增大 Id，记录不同 Id 下磁链变化
  • 适用于精确建模、考虑磁饱和的场景

5. 转动惯量 Js 和摩擦系数 B

方法一：自由减速法（最实用）

  1. 电机加速到已知转速 ω₀
  2. 突然断电，记录速度衰减曲线 ω(t)
  3. 拟合：ω(t) = ω₀ × e^(-B/J × t)
  4. 通过斜率求解 J 和 B

方法二：加速度法

  1. 施加已知转矩 Te（通过 Iq × Kt 计算）
  2. 空载加速，测量角加速度 α = dω/dt
  3. J = Te / α（忽略摩擦时）
  4. 稳态时：B = Te / ω

方法三：阶跃响应辨识

  • 施加阶跃转矩，记录速度响应曲线
  • 机械系统为一阶模型：τ_m = J/B
  • 拟合曲线即可同时获得 J 和 B

6. 角度偏移 e_off、r_off

这是标定（Calibration）过程，不是电机固有参数，每台电机+编码器组合都不同。

标准做法：

  1. 给 d 轴注入固定电流（如 Id=1A，Iq=0），电角度 θ=0
  2. 转子自动对齐到 d 轴方向并锁定
  3. 读取编码器角度值 θ_encoder
  4. e_off = θ_encoder × pn (mod 2π)
  5. r_off = θ_encoder

三、这些参数在 FOC 控制中有什么用

一句话概括：FOC 的本质是数学模型控制电机，模型越准，控制越好。每个参数都是模型中的一个变量。

1. 电流环（最内层，~20kHz）—— 需要 Rs、Ld、Lq、flux

电流环还需要前馈解耦：

参数 用在哪
Rs PI 积分增益 + 电压补偿
Ld d 轴 PI 比例增益 + q 轴解耦前馈
Lq q 轴 PI 比例增益 + d 轴解耦前馈
flux q 轴反电动势前馈补偿

如果 Rs/Ls 不准 → PI 参数不匹配 → 电流响应振荡或迟钝

2. 速度环（中间层，~1kHz）—— 需要 Js、B、Kt

参数 用在哪
Js 速度环 PI 整定，前馈加速度补偿
B 摩擦补偿，稳态误差消除
Kt 转矩 → 电流换算（最关键的桥梁）
wm_acc/dec S 曲线/梯形速度规划的加减速限制

如果 Js 不准 → 速度环增益不匹配 → 加减速超调或响应慢
如果 Kt 不准 → 转矩输出与期望不符 → 力控模式不准

3. 角度/磁场定向 —— 需要 pn、e_off、r_off

FOC 的核心是 Park 变换，必须知道转子磁极的精确位置：

参数 用在哪
pn 机械角度 → 电角度换算
e_off 编码器零点与电角度零点的偏差补偿
r_off 编码器零点与机械零点的偏差补偿

如果 e_off 偏了几度 → Park 变换角度错误 → d/q 轴耦合 → 出力下降、效率变差、电流振荡
这是 FOC 调试中最常见的问题：标定角度不准，一切白搭。

4. 无感观测器 —— 需要 Rs、Ls、flux、Ldif

无感算法（NLOB、ALOB、SMO、ESMO）本质上都在做同一件事——用电机数学模型反推转子位置：

观测器已知 V 和 I，用 Rs、Ls 算出反电动势，再从反电动势提取 θ 和 ω：

参数 用在哪
Rs 从电压中扣除电阻压降
Ls 从电压中扣除电感压降
flux 反电动势幅值的归一化
Ldif 高频注入法（HFI），利用 Ld≠Lq 的凸极效应估计低速位置

如果 Rs 偏了 20% → 观测器反电动势估计错误 → 位置偏差 → 失步

5. V/F 模式 —— 参数要求最低

开环 force_volt_mode 实际只用到 pn：

V/F 模式不依赖 Rs、Ls、flux 等参数，这正是它的优势——用于：

  • 初始验证硬件是否正常
  • 拖动电机辨识 flux（反电动势法）
  • 编码器标定前的基础旋转

总结：参数 → 功能映射

参数是 FOC 数学模型的”灵魂”，参数准确度直接决定控制性能上限。

四、典型参数辨识流程

很多 FOC 方案（包括本项目架构）支持上电自动辨识：依次执行 Rs → Ls → flux → 编码器标定，把结果写入 Flash，量产时只需跑一次标定程序即可。这就是 calibrat_mode 和 idpm_t（参数辨识结构体）的设计目的。

参数辨识的精度直接决定了 FOC 控制器的性能上限。工程中通常先用粗略参数跑起来，再通过在线辨识或离线精测迭代优化。
