5. volatile 有什么作用？ / What does volatile do?

volatile 告诉编译器：这个变量的值可能在程序正常控制流之外被改变，不要对它做任何优化。

English: The volatile qualifier tells the compiler: “This variable’s value can change outside the normal program flow — do not cache it in a register, do not eliminate seemingly redundant reads/writes, and do not reorder accesses to it.”

编译器通常会把频繁访问的变量缓存在寄存器中（register caching）、消除”看似多余”的读写（dead store elimination）、重排指令顺序（instruction reordering）。volatile 强制编译器每次都从内存中重新读取该变量。

三大必须使用的场景 / Three must-use scenarios：

  1. **ISR 与主循环共享的变量 (Variables shared between ISR and main loop)**：ISR 中设的 flag，主循环中检测——如果不加 volatile，编译器可能认为 flag 从未被修改，直接优化成死循环。Without volatile, the compiler may assume the flag never changes and optimize the check into an infinite loop.
  2. **硬件寄存器 (Hardware registers)**：外设寄存器的值随时可能因硬件事件改变。The value of a peripheral register can change at any time due to hardware events.
  3. **多线程 / RTOS 共享变量 (Variables shared across RTOS tasks/threads)**：另一个任务可能随时修改该变量。Another thread or task may modify the variable at any time.

注意 / Note： volatile 不提供原子性（atomicity），也不提供内存屏障（memory barrier）。需要原子操作时还需要配合关中断（disable interrupts）或专用原子指令。

6. const volatile 同时使用是什么意思？ / What does const volatile mean together?

  • const：对程序员来说是只读的，代码中不允许修改。The code must not write to this variable.
  • volatile：值可能在代码控制之外改变，编译器不要优化。The value may change outside the program’s control; compiler must re-read every time.

English: When used together, const volatile means: “My code is not allowed to write to this variable, BUT its value can still change (due to hardware), so the compiler must always read from the actual memory address.” The classic use case is a read-only hardware status register.

7. static 关键字在 C 中的三种用法 / Three uses of static in C

位置 (Context) 含义 (Meaning)
修饰局部变量 (On a local variable) 变量存放在静态区（而非栈），函数退出后值保留。The variable is placed in static memory (not the stack); its value persists across function calls. Lifetime = entire program.
修饰全局变量 (On a global variable) 将可见范围限制在本文件内（内部链接 internal linkage）。Restricts visibility to the current translation unit — other files cannot access it via extern.
修饰函数 (On a function) 将函数可见范围限制在本文件内（file-private function）。Makes the function private to the file — invisible to the linker from other files.

  1. **定义中断向量表 (Define the vector table)**：第一项是初始栈指针 MSP (Initial Stack Pointer)，第二项是 Reset_Handler 入口地址，后面依次是各中断/异常的处理函数地址 (handler addresses for all interrupts and exceptions)。
  2. **Reset_Handler 执行初始化 (Initialization in Reset_Handler)**：
    • 将 .data 段从 Flash 复制到 RAM（Copy initialized global variables from Flash to RAM）
    • 将 .bss 段清零（Zero out uninitialized global variables）
    • 可选：初始化 FPU、调用 SystemInit() 配置时钟（Optionally init FPU, call SystemInit() for clock config）
  3. **跳转到 main() (Branch to main())**：调用 C 运行时入口 __main（ARM）或直接调用 main()。
  4. **提供默认中断处理 (Provide default handlers)**：未实现的中断指向一个死循环的 Default_Handler (infinite loop)，方便调试 — so unhandled interrupts are caught during debugging instead of causing undefined behavior.

22. MCU 上电后的完整启动流程 / MCU boot sequence after power-on

以 ARM Cortex-M 为例 / Using ARM Cortex-M as an example:

  1. **上电/复位 (Power-on / Reset)**：硬件从中断向量表第一项读取 MSP 初始值并加载到 SP 寄存器。The hardware loads the initial MSP value from the first entry of the vector table into the SP register.
  2. **取 Reset 向量 (Fetch Reset Vector)**：从向量表第二项（偏移 offset 0x04）读取 Reset_Handler 地址并跳转执行。The processor reads the Reset_Handler address from offset 0x04 and branches to it.
  3. Reset_Handler（在 Startup 文件中 / in the startup file）：
    • 复制 .data 段 Flash → RAM (Copy .data section)
    • 清零 .bss 段 (Zero-fill .bss section)
    • 调用 SystemInit() 配置时钟树、Flash 等待周期 (Configure clock tree, Flash wait states)
  4. **调用 main() (Call main())**：进入 C 运行时，C++ 全局构造函数执行 (global constructors run if C++)，最终进入 main()。
  5. **在 main() 中 (Inside main())**：初始化外设 (init peripherals: GPIO, UART, timers, etc.)，启动 RTOS（如果有 if applicable），进入主循环 (enter the super loop)。

Key interview point: The very first thing the CPU does after reset is load the stack pointer, then jump to Reset_Handler — it does NOT go straight to main().

23. 什么是 PWM？ / What is PWM?

PWM（Pulse Width Modulation，脉冲宽度调制）通过改变方波的占空比（duty cycle）来等效控制输出”平均值”。

English: PWM generates a square wave with a controllable duty cycle. The average output voltage is proportional to the duty cycle: 0% = always LOW, 50% = half HIGH half LOW, 100% = always HIGH.

用途 / Applications： LED 亮度调节 (LED dimming)、电机速度控制 (motor speed control)、舵机角度控制 (servo positioning)、DAC 替代 (poor man’s DAC via low-pass filtering)、开关电源 (switch-mode power supply).

24. 什么是 DMA？ / What is DMA and why does it matter?

DMA（Direct Memory Access，直接内存访问）允许外设与内存之间直接传输数据，不需要 CPU 介入每个字节的搬运。

English: DMA allows peripherals to transfer data directly to/from memory without the CPU handling each byte. The CPU configures the DMA controller (source address, destination address, length, direction), then is free to do other work. When the transfer completes, the DMA raises an interrupt to notify the CPU.

典型场景 / Typical use cases： ADC 连续采样 (continuous ADC sampling into a buffer)、UART 收发大块数据 (bulk UART TX/RX)、SPI 与 Flash 之间的数据传输 (SPI ↔ Flash)、内存到内存的拷贝 (memory-to-memory copy).

25. 电机的力矩由电压还是电流决定？ / Is motor torque determined by voltage or current?

电流 (Current)。 电机力矩（扭矩 torque）T = K_t × I，其中 K_t 是力矩常数（torque constant），I 是电流（current）。

English: Torque is proportional to current. Voltage determines the maximum speed (it must overcome back-EMF plus resistive drop). A simple way to think about it: voltage controls how fast it can spin; current controls how much force it can produce.

这就是为什么电机驱动器通常是电流控制（current control / torque mode）或通过 PWM 间接控制电流。堵转时（stall condition）电流最大（反电动势 back-EMF 为零），力矩也最大，但可能烧毁电机或驱动器——所以需要限流保护（current limiting）。

四、中断与系统机制 (Interrupts & System Mechanisms)

中断是嵌入式实时响应的核心，也是 bug 最密集的区域。
Interrupts are the heart of real-time response — and the most bug-prone area in embedded systems.

26. ISR 是什么？为什么它很重要？ / What is an ISR and why does it matter?

ISR（Interrupt Service Routine，中断服务程序）是当硬件或软件中断发生时，CPU 自动跳转执行的一段代码。

English: An ISR is a special function that the CPU automatically executes in response to a hardware or software interrupt. When an external event occurs (e.g., a timer overflow, a button press, or data arriving on a UART), the processor suspends its current task, saves context, and jumps to the corresponding ISR.

重要性 / Why it matters：

  • **实时响应 (Real-time responsiveness)**：外设事件无需 CPU 轮询（polling），中断发生后硬件自动通知 CPU，延迟极低。Peripherals notify the CPU instantly instead of the CPU wasting cycles polling.
  • **效率 (Efficiency)**：CPU 在没有事件时可以做其他工作甚至进入低功耗模式（sleep mode），而不是死等。The CPU is free to do useful work or enter low-power sleep between events.
  • **确定性 (Determinism)**：RTOS 的任务调度、时间片切换等核心机制都建立在定时器中断之上。RTOS scheduling and context switching are built on top of timer interrupts.

编写 ISR 的关键原则 / Key rules for writing ISRs：

  • 尽量短小（keep it short）：清标志位、搬数据到 buffer、设 flag，把重活留给主循环或任务。Do the minimum work — clear the flag, copy data to a buffer, set a flag — and defer heavy processing to the main loop or an RTOS task.
  • 不要在 ISR 里调用 printf、malloc 等不可重入函数（non-reentrant functions）。
  • 共享数据加 volatile。Shared variables accessed by both the ISR and main code must be declared volatile.
  • 在 ARM Cortex-M 中，中断优先级和嵌套由 NVIC（Nested Vectored Interrupt Controller）管理，高优先级中断可以打断低优先级中断（preemption）。

27. SysTick 和 PendSV 详解 / SysTick and PendSV explained

SysTick — 系统节拍定时器 (System Tick Timer)

SysTick 是 ARM Cortex-M 内核自带的一个 24 位递减计数器（24-bit down counter），计到零时产生 SysTick 异常（exception）。

English: SysTick is a 24-bit countdown timer built into every Cortex-M core. When it reaches zero, it fires the SysTick exception, reloads, and starts counting down again — providing a periodic “heartbeat” for the system.

核心用途 / Primary uses：

  • **RTOS 时间基准 (RTOS tick source)**：FreeRTOS 的 xTaskIncrementTick() 就在 SysTick 中断里被调用。每次 tick 到来时，RTOS 检查是否有更高优先级的任务就绪、是否有任务超时需要唤醒。FreeRTOS calls xTaskIncrementTick() inside the SysTick handler — on each tick the kernel checks for ready tasks, timeouts, and delayed tasks that need waking.
  • **HAL 延时 (HAL delay)**：HAL_Delay() 依赖 SysTick 递增一个全局毫秒计数器 uwTick。HAL_Delay() works by incrementing a global millisecond counter uwTick in SysTick.
  • **系统心跳 (System heartbeat)**：看门狗喂狗、LED 闪烁、软件定时器等都可以基于 SysTick。Watchdog feeding, LED blinking, and software timers can all hang off the SysTick tick.

PendSV — 可悬起的系统服务调用 (Pendable Service Call)

PendSV 是一个可以被”挂起”（pend）的异常，由软件手动触发，且被设置为最低优先级。

English: PendSV (Pendable Service Call) is a software-triggered exception that is intentionally configured at the lowest priority. It is the mechanism RTOS kernels use to perform context switching (task switching) safely.

为什么不在 SysTick 里直接做上下文切换？ / Why not context-switch directly in SysTick?

这是面试关键问题。If you context-switch inside SysTick (which runs at a relatively high priority), and a context switch takes time, it could delay other time-critical interrupts. The solution:

  1. SysTick 中断触发 → RTOS 判断”需要切换任务” → 设置 PendSV 挂起位 (set the PendSV pending bit)
  2. SysTick 退出 → 所有更高优先级中断先处理完
  3. CPU 空闲时 → PendSV 作为最低优先级异常被执行 → 在 PendSV Handler 里完成上下文切换

English — the key flow:

  1. SysTick fires → RTOS decides a context switch is needed → pends PendSV (writes to ICSR register)
  2. SysTick returns → any higher-priority interrupts run first
  3. When nothing else is pending → PendSV executes at the lowest priority → performs the actual context switch (save current task’s registers to its stack, load next task’s registers)

这样做的好处 / Benefits：

  • 上下文切换永远不会打断任何 ISR（因为 PendSV 优先级最低）。Context switch never preempts any ISR — it only runs when nothing else needs the CPU.
  • 中断响应延迟不会因为任务切换而增大。Interrupt latency is not increased by task switching overhead.
  • 这就是为什么几乎所有 Cortex-M RTOS（FreeRTOS、Zephyr、RT-Thread）都将 PendSV 配置为最低优先级。This is why virtually every Cortex-M RTOS sets PendSV to the lowest priority.

29. 什么是看门狗？ / What is a Watchdog Timer?

看门狗定时器（Watchdog Timer, WDT）是一个硬件倒计时器，软件必须在超时前定期”喂狗”（kick / feed the watchdog），否则触发系统复位。

English: A watchdog timer is a hardware countdown timer that resets the system if the software fails to refresh (“kick” or “feed”) it before it expires. It protects against software lockups, runaway code, and unresponsive states.

注意事项 / Important note： 不要在 ISR 中喂狗——如果主循环死了但 ISR 还在跑，看门狗就失去了意义。Never feed the watchdog from an ISR — if the main loop is stuck but interrupts are still firing, the watchdog won’t catch the failure.

30. 嵌入式系统中的低功耗设计要点 / Low-Power Design in Embedded Systems

  • **睡眠模式 (Sleep modes)**：Sleep → Stop → Standby。层级越深越省电，唤醒越慢。Deeper modes save more power but take longer to wake up.
  • **时钟门控 (Clock gating)**：不用的外设关掉时钟。Disable clocks to unused peripherals.
  • **降频 (Frequency scaling)**：系统时钟降到满足需求的最低频率。Run the system clock at the minimum frequency that meets requirements.
  • **IO 状态 (Pin state management)**：未使用引脚不要悬空（floating）——设为模拟输入或带上/下拉。Set unused pins to analog input or enable internal pull-up/pull-down.
  • **DMA 替代轮询 (DMA instead of polling)**：CPU 可以在等待数据时睡眠。The CPU can sleep while DMA handles data transfer.
  • **低功耗外设 (Low-power peripherals)**：低功耗定时器 (LPTIM)、低功耗 UART (LPUART) 等可在睡眠模式下工作。Some peripherals (LPTIM, LPUART) can operate during sleep modes.

32. I2C 为什么要加上拉电阻？ / Why does I2C need pull-up resistors?

I2C 的 SDA 和 SCL 都是开漏（Open-Drain）输出：设备只能把线拉低（输出 0），但不能主动驱动为高。释放总线后，需要上拉电阻把线拉回高电平。

English: I2C uses open-drain (or open-collector) outputs — a device can only pull the line LOW but cannot actively drive it HIGH. When a device releases the line, a pull-up resistor is needed to pull it back to the HIGH (idle) state.

原因与好处 / Reasons and benefits：

  • **线与逻辑 (Wired-AND logic)**：任何一个设备拉低，总线就是低。只有所有设备都释放，总线才回到高。这是总线仲裁（bus arbitration）和时钟延展（clock stretching）的基础。Any device pulling LOW wins — the bus only goes HIGH when ALL devices release. This enables multi-master arbitration and clock stretching.
  • **电平兼容 (Voltage level compatibility)**：上拉到不同电压可以适配 3.3V 或 5V 系统。Pull up to different voltages for 3.3V or 5V system compatibility.
  • **避免短路 (Avoid short circuits)**：如果用推挽输出（push-pull），两个设备同时一个输出高、一个输出低，就会短路。With push-pull outputs, simultaneous HIGH and LOW from two devices would cause a short circuit.

上拉电阻的选择 / Choosing the resistor value： 常用 4.7kΩ（标准模式 standard mode）到 1kΩ（高速或长线缆 high-speed or long cables）。阻值太大上升沿太慢（too slow rise time），太小功耗太大（excessive power consumption）。

33. 终端电阻是什么？有什么作用？ / What is a termination resistor and what does it do?

终端电阻（Termination Resistor）加在传输线的末端，匹配传输线的特性阻抗（characteristic impedance），防止信号反射（signal reflection）。

English: A termination resistor is placed at the end of a transmission line to match its characteristic impedance. Without it, signals reflect off the unterminated end, bounce back, and interfere with the original signal — causing ringing, overshoot, and data corruption.

常见应用 / Common applications：

RS-485 120Ω 同 CAN，长距离差分总线。Same as CAN — long-distance differential bus.

关键点 / Key point： 终端电阻通常只加在总线的物理两端，中间节点不加。Termination resistors go only at the two physical ends of the bus — never at intermediate nodes.

34. 总线仲裁是干什么的？ / What is bus arbitration?

在多主（Multi-Master）总线中，如果两个或多个主机同时发起通信，就会发生冲突。总线仲裁的作用是在不破坏数据的前提下，决定哪个主机获得总线控制权。

English: Bus arbitration is the mechanism that resolves collisions when multiple masters try to transmit on a shared bus simultaneously. It determines which master gains control of the bus — without corrupting any data.

I2C 仲裁机制 / I2C arbitration： 利用线与（wired-AND）特性——多个主机同时发送时，发低电平的那个”赢”。每个主机在发送每一位后都会回读总线状态（read-back），如果自己发的是高但总线实际是低，说明有其他主机在发低，自己就退出（backs off），不会破坏赢家的数据。

Each master monitors the bus after each bit. If a master sends HIGH but reads LOW, it knows another master is driving the bus — it loses arbitration and stops transmitting, without corrupting the winner’s data.

CAN 仲裁机制 / CAN arbitration： CAN 帧以仲裁域（Arbitration Field / ID）开头，ID 数值越小优先级越高。所有节点同时发送，低 ID 的自然”压过”高 ID 的（dominant bit wins over recessive bit），高 ID 节点检测到冲突后主动退出。整个过程无数据丢失（non-destructive arbitration）。

CAN frames begin with an ID field. Lower ID = higher priority. All nodes transmit simultaneously; the dominant bit (0) wins over the recessive bit (1). Higher-ID nodes detect the mismatch and back off — zero data loss.

六、RTOS 与多任务 (RTOS & Multitasking)

35. 什么是 RTOS？ / What is an RTOS and why use it?

RTOS（Real-Time Operating System，实时操作系统）在多个任务之间进行调度，保证高优先级任务在规定时间内得到响应。

English: An RTOS is a lightweight operating system that schedules multiple tasks based on priority, guaranteeing that the highest-priority ready task gets the CPU within a bounded time.

**裸机的问题 / Problem with bare-metal (super loop)**：所有逻辑在一个大 while(1) 中顺序执行，如果某个操作阻塞或耗时长，其他功能就被延迟。Everything runs sequentially in one loop; a long-running operation delays everything else.

RTOS 的好处 / Benefits of RTOS： 任务独立 (independent tasks)、优先级抢占 (preemptive priority scheduling)、阻塞 API (blocking APIs — a task waiting on a semaphore doesn’t waste CPU)、更好的代码组织 (better code modularity).

常见 RTOS / Common RTOS： FreeRTOS, Zephyr, RT-Thread, ThreadX (Azure RTOS).

59. 分析中断优先级反转场景 / Analyze priority inversion scenario

面试官：解释为什么这段代码可能导致高优先级任务被无限阻塞。 / Explain why the high-priority task may be blocked indefinitely.

分析 / Analysis：

  1. Task_L 获取信号量，开始执行 access_shared_resource()。
  2. Task_H 就绪，抢占 Task_L（优先级更高），尝试获取信号量 → 被阻塞（Task_L 还持有）。
  3. Task_M 就绪。由于 Task_H 被阻塞，Task_M 是当前最高优先级就绪任务 → Task_M 抢占 Task_L 运行。
  4. Task_M 的 cpu_intensive_work() 可以运行很长时间 → Task_L 得不到 CPU → 无法释放信号量 → Task_H 被间接阻塞。

English — the chain:

  1. Task_L holds the semaphore and is running.
  2. Task_H wakes up, tries to take the semaphore → blocked (held by Task_L).
  3. Task_M wakes up. Since Task_H is blocked, Task_M is the highest-priority ready task → it preempts Task_L.
  4. Task_M runs for a long time → Task_L can’t release the semaphore → Task_H is indirectly blocked by Task_M, which has lower priority than Task_H.

这就是优先级反转（Priority Inversion）——高优先级任务被低优先级任务间接阻塞。

修复 / Fix： 使用 Mutex（互斥锁） 替代 Semaphore。Mutex 支持优先级继承（Priority Inheritance）——当 Task_H 阻塞在 Mutex 上时，RTOS 自动将 Task_L 的优先级临时提升到与 Task_H 相同，使 Task_L 不会被 Task_M 抢占，尽快完成并释放 Mutex。

Use a Mutex instead of a Semaphore. Mutexes support priority inheritance — when Task_H blocks on the Mutex, the RTOS temporarily raises Task_L‘s priority to match Task_H, preventing Task_M from preempting Task_L. Task_L finishes quickly and releases the Mutex.
