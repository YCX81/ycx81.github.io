🔧 嵌入式知识：RTOS 任务调度与同步原语

原理

RTOS（实时操作系统） 的核心是调度器（Scheduler），它决定哪个任务在任何时刻占用 CPU。

调度策略：

  • 抢占式（Preemptive）：高优先级任务一就绪就立即夺取 CPU（FreeRTOS 默认）
  • 时间片（Round-Robin）：同优先级任务轮流占用 CPU 一个 tick
  • 协作式（Cooperative）：任务主动让出 CPU，不会被打断（危险：一个任务卡死全系统卡死）

常见误区

❌ 在 ISR 里调用普通 API（非 FromISR 版本）
→ 会导致 crash 或断言失败，ISR 只能用 xSemaphoreGiveFromISR、xQueueSendFromISR 等

❌ 用 vTaskDelay() 做精确周期控制
→ 误差会累积（执行时间叠加进去），必须用 vTaskDelayUntil()

❌ 优先级反转（Priority Inversion）
→ 低优先级任务持有 mutex，中等优先级任务不断抢占，高优先级任务永远等不到 mutex
→ 解决：使用支持优先级继承的 Mutex（FreeRTOS Mutex 自动处理，Binary Semaphore 不支持）

❌ 堆栈溢出（Stack Overflow）
→ 每个任务需要独立栈，过小会静默覆盖相邻内存
→ 开启 configCHECK_FOR_STACK_OVERFLOW，调试期用 uxTaskGetStackHighWaterMark() 测峰值

面试 Q&A

Q：Mutex 和 Binary Semaphore 的区别？
A：Mutex 有所有权概念（只有获取者才能释放）并支持优先级继承；Binary Semaphore 没有所有权，可以由另一个任务或 ISR 释放，适合任务间信号传递。用错会导致优先级反转和死锁。

Q：为什么 ISR 里不能用 vTaskDelay？
A：ISR 运行在中断上下文，没有任务 TCB，调用任何会阻塞调度器的 API 都会导致 crash。ISR 应尽量短，只做”给信号/发队列”，把处理工作交给任务。

Q：FreeRTOS 中如何避免堆碎片化？
A：优先使用静态分配（xTaskCreateStatic、xQueueCreateStatic），或选择 heap_4（best-fit + 合并相邻空闲块），或 heap_5（多个非连续内存区域）。

Q：configTICK_RATE_HZ 设多少合适？
A：1000 Hz（1ms tick）是常用选择，兼顾时间精度和调度开销。对于电机控制等需要 < 1ms 精度的场景，不要依赖 tick，改用硬件定时器中断直接触发任务。

  1. semaphore /ˈsɛməfɔːr/ — 信号量 技术

    A counting semaphore tracks the number of available DMA channels.

  2. mutex /ˈmjuːtɛks/ — 互斥锁 技术

    Always acquire the mutex before accessing the shared sensor buffer.

  3. preemption /priˈɛmpʃən/ — 抢占（调度） 技术

    Preemption allows a high-priority task to immediately take control of the CPU.

  4. scheduler /ˈskɛdʒʊlər/ — 调度器 技术

    The RTOS scheduler runs every tick and selects the highest-priority ready task.

  5. deadlock /ˈdɛdlɒk/ — 死锁 技术

    Acquiring two mutexes in different orders across tasks can cause a deadlock.

  6. starvation /stɑːrˈveɪʃən/ — （任务）饥饿 技术

    Without aging, low-priority tasks may suffer starvation in a preemptive system.

  7. spinlock /ˈspɪnlɒk/ — 自旋锁 技术

    A spinlock is efficient only when the lock is held for very short periods.

    FreeRTOS uses a preemptive scheduler by default.

  9. priority /praɪˈɒrɪti/ — 优先级 技术/日常
