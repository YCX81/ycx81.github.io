1.3 负载特性分析

从T-N曲线和最大过载曲线可以判断：

  • 恒转矩区：低速段（<120rpm），可维持12N·m峰值转矩
  • 恒功率区：120~280rpm，转矩随转速上升而下降
  • 过载时间限制：12N·m仅能维持28秒，4N·m为额定连续

结论：这是一个典型的机器人关节电机，主要工作在低速大扭矩场景，以恒转矩工况为主，偶尔需要短时过载。

1.4 控制精度要求

从参数表观测量可知系统需要：

  • 位置精度：14bit编码器 → 机械角度分辨率 0.022°，经减速器后负载端分辨率约 0.0028°
  • 速度精度：从示波器功能和速度滤波参数(spd_filt_gain=0.1)可知速度环带宽不高
  • 电流精度：ADC采样零偏校准(adc1Offset, adc2Offset)，电流滤波(cur_filt_gain=0.9)

1.5 有感 vs 无感

CyberGear使用AS5047P磁编码器（有感方案）：

  • 14bit单圈绝对值，SPI通信
  • 支持多圈计数（rotation参数）
  • 无需无感观测器，启动和低速性能有保证
  • 弱磁电流参数 I_FW_MAX 默认为0（未启用弱磁）

对比选型参考：

  • STM32F405/407：同为Cortex-M4 168MHz，生态更成熟，ST FOC库(MCSDK)可直接用
  • STM32G431：Cortex-M4 170MHz，集成运放+比较器，电流采样更灵活
  • STM32H743/750：Cortex-M7 480MHz，性能过剩但适合复杂算法（如无感高频注入）

GD32F303选择的核心考量：成本。对于量产消费级产品，国产MCU的价格优势明显。

2.2 驱动芯片：6EDL7141（英飞凌）

6EDL7141是英飞凌的智能三相栅极驱动芯片，功能非常丰富：

  • 集成三相半桥驱动：支持高低侧N-MOSFET驱动
  • 自适应死区控制：可编程死区时间，减少失真
  • 内置电荷泵：无需自举电容（简化设计）
  • SPI配置接口：死区时间、驱动电流、保护阈值均可通过SPI配置
  • 集成电流放大器：支持底端电阻采样
  • 过流/过温/欠压保护：硬件级保护

对比常见方案：

  • DRV8301/8302（TI）：经典方案，集成功率MOSFET或外部MOSFET驱动
  • IR2136（英飞凌旧款）：简单可靠但功能少
  • FD6288（峰岹）：国产低成本方案

6EDL7141的选择体现了高集成度+高性能的设计理念，非常适合紧凑型关节电机。

第五章 调试流程详解

5.1 开环验证（首次上电必做）

目的：验证硬件接线正确，电机可以旋转

验证项目：

但CyberGear实际用的 Kp=0.025, Ki=0.0258，远小于理论值。这说明：

  1. 先只加Kp，从小到大增加，直到电流响应有轻微振荡
  2. 降低Kp到振荡值的60~70%
  3. 逐步增加Ki，直到稳态误差消除
  4. 用正弦波电流指令测试跟踪性能（CyberGear调试器的”正弦化自动测试”功能）
  5. 观察id和iq的示波器波形，确认无振荡、跟踪准确

5.4 闭合速度环调参

CyberGear默认参数：spd_kp = 2.0, spd_ki = 0.021

速度环带宽应低于电流环带宽的1/5~1/10：

调参步骤：

  1. 先设定较小的速度指令（如1 rad/s）
  2. 从小Kp开始，增大直到速度响应快速但不振荡
  3. 加入Ki消除稳态误差
  4. 测试阶跃响应和负载突变响应
  5. 调整 spd_filt_gain（默认0.1，低通滤波）平衡噪声和响应速度

5.5 弱磁调试

CyberGear的 I_FW_MAX 默认为0（未启用弱磁）。

弱磁的目的：当电机转速较高时，反电势接近母线电压，电流环无法继续控制。此时通过注入负的id电流来削弱磁场，提高可用电压空间。

对于CyberGear这种低速大扭矩关节电机，一般不需要弱磁。

第七章 性能优化

7.1 中断耗时优化

FOC主循环需在50μs内完成（20kHz → 周期50μs），典型执行时间约10~30μs。

优化手段：

  • 查表法替代三角函数：sinf/cosf用1024点查找表+线性插值
  • 避免除法：预计算 1/Udc, 1/√3 等常数
  • 利用Cortex-M4 FPU：确保编译器开启硬件浮点（-mfpu=fpv4-sp-d16）
  • 合理使用DMA：ADC采样结果自动搬运，减少CPU干预

7.2 死区补偿

逆变器的死区会导致电压失真（尤其在低电流过零点附近），表现为电流波形的零交叉畸变和6次谐波转矩脉动。

6EDL7141的自适应死区功能可以在硬件层面缓解这个问题。

7.3 谐波抑制

  • 电流采样同步：确保ADC在PWM中心采样，避免开关噪声
  • 电流滤波：CyberGear的 cur_filt_gain = 0.9 是一阶低通：
  • 速度滤波：spd_filt_gain = 0.1（更重的滤波，因为速度信号噪声更大）

第九章 标幺值系统

9.1 问题的根源

实验室调试时，你直接用物理量编程：

这在量产中会导致严重问题：

  1. 同一套PI参数无法跨平台复用。换一个电压等级的电机（24V→48V），所有PI参数必须重新整定，因为被控对象的增益变了。

  2. ADC量程利用率问题。不同批次电机的额定电流不同（6.5A±10%），固定增益的采样电路对某些电机利用率高、某些低，分辨率不一致。

  3. 定点化困难。物理量的数值范围跨度巨大（电流0~23A，电压0~28V，角度0~2π，角速度0~3000 rad/s），用统一的Q格式很难兼顾精度和范围。

  4. 控制器饱和行为不一致。PI输出限幅用物理量时，不同母线电压下饱和阈值不同，控制行为也不同。

9.2 什么是标幺值（Per-Unit, pu）

标幺值 = 物理量 / 基准值，结果是一个无量纲的归一化数。

选定基准值后，所有物理量都在 [-1, 1] 或 [0, 1] 范围内表示。

9.3 FOC中的基准值选取

基准值的选取有一套内在逻辑，不是随便选的。核心原则是让基准量之间满足电路基本关系（欧姆定律、功率公式等），这样标幺化后的方程形式不变。

第十章 定点化实现

10.1 为什么量产要考虑定点化

因素 浮点(float) 定点(Q15/Q31)
成本 Cortex-M4/M7（有FPU），单价更高 Cortex-M0/M0+/M3，单价低50%以上
功耗 FPU运算功耗较高 纯整数运算，功耗低
确定性 float运算周期不完全确定 整数运算周期固定，中断时序确定
精度 23bit尾数（float32） Q15=15bit，Q31=31bit

CyberGear用的GD32F303有FPU，直接用float是合理的。 但如果量产目标是降本到Cortex-M0级别MCU（如GD32E230，单价<2元），定点化就是必须的。
