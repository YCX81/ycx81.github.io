🔧 嵌入式知识：链接脚本与内存布局

一、程序内存布局基础

一个典型的嵌入式程序在 Flash 和 RAM 中的布局如下：

各段含义：

段名 存储位置 内容 特点
.text Flash 代码（函数指令） 只读，可共享
.rodata Flash 常量字符串、const 全局变量 只读
.data RAM（初值在 Flash） 有初值的全局/静态变量 启动时从 Flash 复制
.bss RAM 无初值（或初值为 0）的全局/静态变量 启动时清零
heap RAM malloc / new 的动态内存 向高地址增长
stack RAM 函数调用栈、局部变量 向低地址增长

二、链接脚本（Linker Script）

链接脚本（.ld 文件）是 GNU ld 的配置文件，告诉链接器：

  • 芯片的 Flash/RAM 起始地址和大小
  • 各段放在哪里
  • 哪些符号要导出给启动代码

典型 STM32 链接脚本结构

关键符号含义

符号 含义
_sdata .data 段在 RAM 中的起始地址
_edata .data 段在 RAM 中的结束地址
_sidata .data 段初值在 Flash 中的地址（AT>FLASH 自动生成）
_sbss .bss 段起始
_ebss .bss 段结束
_estack 初始栈顶（MSP 寄存器初值）

四、常见陷阱

1. .data 段没有被复制

如果 _sidata / _sdata / _edata 符号名与启动文件不匹配，全局变量初值为 0 而非预期值——这是非常隐蔽的 bug。

2. 栈溢出

栈从高地址向低地址增长，若栈大小不够，会覆盖 .bss 或 .data，导致随机内存损坏。在 .bss 末尾放一个”栈水位” canary 可以及时检测。

五、面试常见问法

Q1：.data 和 .bss 的区别？

.data 存有非零初值的全局/静态变量，初值保存在 Flash，启动时复制到 RAM；.bss 存初值为 0 的变量，启动时清零即可，不占 Flash 空间。

Q2：为什么栈要放在 RAM 末尾？

栈向低地址增长，放在 RAM 末尾（最高地址）可以最大化栈的可用空间，且溢出时不会立即损坏其他段。

Q3：链接脚本中 AT> 是什么意思？

AT> 指定段的加载地址（Load Memory Address, LMA）与运行地址（Virtual Memory Address, VMA）不同。.data 的 LMA 在 Flash，VMA 在 RAM，启动代码据此将数据复制过去。

Q4：如何查看最终的内存分配？

使用 arm-none-eabi-size 查看各段大小，使用 arm-none-eabi-objdump -h 查看详细段信息，用 arm-none-eabi-nm --size-sort 找最大的符号。

逐词详解

1. linker /ˈlɪŋkər/

📖 链接器 [技术]
💬 The GNU linker (ld) combines object files and resolves symbol references to produce the final ELF binary.

2. relocate /ˌriːloʊˈkeɪt/

📖 重定位 [技术]
💬 When a shared library is loaded at a different address, the loader must relocate all absolute references.

3. symbol /ˈsɪmbəl/

📖 符号（链接器符号） [技术]
💬 Undefined symbols at link time usually mean you forgot to add the source file to the build system.

4. segment /ˈsɛɡmənt/

📖 段（ELF Program Header 中的概念） [技术]
💬 The LOAD segment in an ELF file tells the loader which data to copy into memory at startup.

5. section /ˈsɛkʃən/

📖 节（ELF Section 概念） [技术]
💬 You can place a function in a specific section using __attribute__((section(".fast_code"))) in GCC.

6. executable /ɪɡˈzɛkjʊtəbl/

📖 可执行文件 [技术]
💬 The final executable is generated after the compiler, assembler, and linker all complete successfully.

7. binary /ˈbaɪnəri/

📖 二进制文件；固件镜像 [技术]
💬 Use arm-none-eabi-objcopy -O binary to convert the ELF to a raw binary for flashing.

8. flash /flæʃ/

📖 闪存；烧录 [技术]
💬 The linker script must accurately reflect the Flash size or you risk overflowing into unintended regions.

9. origin /ˈɒrɪdʒɪn/

📖 起始地址 [技术]
💬 Set ORIGIN = 0x08000000 for the Flash region on most STM32 devices.

10. allocation /ˌæləˈkeɪʃən/

📖 分配 [技术]
💬 Static allocation at compile time is preferred over dynamic allocation in safety-critical firmware.

11. virtual /ˈvɜːrtʃuəl/

📖 虚拟的（VMA，运行时地址） [技术]
💬 The virtual address of .data is in RAM, but its load address (LMA) is in Flash.

12. physical /ˈfɪzɪkəl/

📖 物理的（实际加载地址） [技术]
💬 The physical address is where data resides in Flash; the virtual address is where the CPU accesses it at runtime.

13. initialize /ɪˈnɪʃəlaɪz/

📖 初始化 [技术]
💬 The Reset_Handler initializes .data by copying from Flash and zeros out .bss before calling main.

14. zero-initialize /ˈzɪərəʊɪˈnɪʃəlaɪz/

📖 零初始化 [技术]
💬 C standard guarantees that global variables are zero-initialized before program startup.

15. canary /kəˈneəri/

📖 金丝雀（栈溢出检测值） [技术]
💬 GCC’s stack protector inserts a canary value before the return address to detect stack overflows.

16. scatter /ˈskætər/

📖 分散加载文件（ARM Keil 术语） [技术]
💬 ARM’s scatter file is the Keil/armcc equivalent of the GNU linker script.

17. region /ˈriːdʒən/

📖 内存区域 [技术]
💬 Define separate MEMORY regions for internal Flash, external QSPI Flash, and SRAM in the linker script.

18. placement /ˈpleɪsmənt/

📖 放置；布局（段的位置安排） [技术]
💬 Careful placement of time-critical code in SRAM can reduce execution latency compared to Flash.

19. objdump /ˈɒbdʒdʌmp/

📖 GNU 目标文件分析工具 [技术]
💬 Run arm-none-eabi-objdump -h firmware.elf to inspect section sizes and addresses.

20. disassemble /ˌdɪsəˈsɛmbəl/

📖 反汇编 [技术]
💬 Disassemble the HardFault handler to trace back the faulting instruction from the LR register.

21. map file /mæp faɪl/

📖 链接映射文件（.map） [技术]
💬 The linker map file shows exactly where each function and variable was placed in memory.

22. relocation /ˌriːloʊˈkeɪʃən/

📖 重定位（链接时地址修正） [技术]
💬 Relocation entries in the object file tell the linker where to patch absolute addresses.

23. address space /ˈædrɛs speɪs/

📖 地址空间 [技术]
💬 Cortex-M has a 4 GB flat address space; peripherals, Flash, and RAM are all memory-mapped into it.

24. contiguous /kənˈtɪɡjuəs/

📖 连续的（地址连续） [技术]
💬 A DMA transfer requires the source and destination buffers to occupy contiguous memory regions.

25. padding /ˈpædɪŋ/

📖 填充字节（对齐用） [技术]
💬 The linker inserts padding bytes between sections to satisfy alignment constraints.

26. corrupt /kəˈrʌpt/

📖 损坏，破坏（数据/内存） [技术]
💬 A stack overflow that corrupts the heap will cause silent, hard-to-reproduce failures.

27. overwrite /ˌoʊvərˈraɪt/

📖 覆写，覆盖（内存内容） [技术]
💬 An out-of-bounds array write can overwrite adjacent variables or even the return address.

28. epilogue /ˈepɪlɒɡ/

📖 函数尾声（恢复栈帧、弹出寄存器） [技术]
💬 The function epilogue restores the frame pointer and stack pointer before returning.

29. prologue /ˈproʊlɒɡ/

📖 函数开场（保存寄存器、建立栈帧） [技术]
💬 The function prologue pushes callee-saved registers and allocates space for local variables.

30. frame pointer /freɪm ˈpɔɪntər/

📖 帧指针 [技术]
💬 Omitting the frame pointer (-fomit-frame-pointer) frees one register but complicates stack unwinding.

31. stack pointer /stæk ˈpɔɪntər/

📖 栈指针（SP = R13 on Cortex-M） [技术]
💬 On Cortex-M, the reset vector table’s first entry initializes the Main Stack Pointer (MSP).

32. heap fragmentation /hiːp ˌfræɡmɛnˈteɪʃən/

📖 堆碎片化 [技术]
💬 Repeated malloc/free of variable-size blocks leads to heap fragmentation over time.

33. pool allocator /puːl ˈæləkeɪtər/

📖 内存池分配器 [技术]
💬 A pool allocator pre-allocates fixed-size blocks, eliminating fragmentation and making allocation O(1).

34. watermark /ˈwɔːtərmɑːrk/

📖 水位线（栈或内存使用量） [技术]
💬 Fill the stack with 0xDEADBEEF and check the watermark at runtime to estimate maximum stack usage.

35. stripping /ˈstrɪpɪŋ/

📖 去除调试信息（strip） [技术]
💬 Strip debug symbols from the release binary to reduce Flash size and protect IP.

36. objcopy /ˈɒbdʒkɒpi/

📖 GNU 目标文件格式转换工具 [技术]
💬 Use objcopy -O ihex firmware.elf firmware.hex to create an Intel HEX file for the programmer.

37. hex file /hɛks faɪl/

📖 Intel HEX 格式烧录文件 [技术]
💬 Most MCU programmers accept Intel HEX or binary format; choose based on your tool chain.

38. bootrom /ˈbuːtrɒm/

📖 芯片厂商固化的启动 ROM [技术]
💬 The STM32 bootrom at 0x1FFF0000 enables DFU mode when BOOT0 is pulled high at reset.

39. scatter loading /ˈskætər ˈloʊdɪŋ/

📖 分散加载（Keil MDK 概念） [技术]
💬 Scatter loading lets you place the vector table in one region and the main code in another.

40. dead code /dɛd koʊd/

📖 死代码（永远不会执行的代码） [技术]
💬 Enable -ffunction-sections -fdata-sections and --gc-sections to eliminate dead code at link time.

41. eliminate /ɪˈlɪmɪneɪt/

📖 消除，去除 [技术/书面]
💬 Use link-time optimization (LTO) to eliminate redundant function calls across translation units.

42. retention /rɪˈtɛnʃən/

📖 保持；数据保留（低功耗场景） [技术/书面]
💬 Declare variables in a .noinit section if they should survive a software reset without re-initialization.

43. persistent /pərˈsɪstənt/

📖 持久的；断电或复位后仍保留 [技术/日常]
💬 Store persistent configuration in a dedicated Flash region with wear-leveling to extend device lifespan.

44. non-volatile /nɒn ˈvɒlətaɪl/

📖 非易失性（断电不丢失） [技术]
💬 EEPROM and Flash are non-volatile memories suitable for storing calibration parameters.

45. manifest /ˈmænɪfɛst/

📖 清单；清晰表明 [技术/书面]
💬 A firmware manifest lists the version, CRC, and expected load address of each image in the OTA package.

46. delineation /dɪˌlɪniˈeɪʃən/

📖 划定；明确描述 [书面]
💬 Clear delineation of Flash regions for bootloader, application, and configuration avoids accidental overwrites.

47. contend /kənˈtɛnd/

📖 竞争；争夺（资源） [技术/书面]
💬 When both CPU and DMA contend for the same SRAM bus, DMA can stall the CPU pipeline.

48. segregate /ˈsɛɡrɪɡeɪt/

📖 隔离；将不同类型数据分开存放 [技术/书面]
💬 Segregate safety-critical data and normal application data into separate MPU-protected memory regions.

49. reside /rɪˈzaɪd/

📖 驻留；存放在某个地址 [技术/书面]
💬 The interrupt vector table must reside at the base of Flash unless the VTOR register is configured.

50. designate /ˈdɛzɪɡneɪt/

📖 指定；划定为 [技术/书面]
💬 Designate a fixed 4 KB region at the end of Flash for non-volatile configuration storage.

💬 地道表达

1. “lay the groundwork”

含义： 奠定基础；打好地基
场景： 技术讨论中谈到基础架构设计、前期准备

📌 出处： The Economist, 2023

“A well-crafted linker script lays the groundwork for every subsequent firmware component — get it wrong and everything built on top will be unstable.”

2. “under the hood”

含义： 深入内部，探究底层机制
场景： 解释某个功能背后实际发生了什么

📌 出处： NYT, 2023

“The HAL makes memory management look simple, but under the hood the linker script and startup code are doing the heavy lifting.”

3. “set in stone”

含义： 一成不变，不可更改；已定死的规则
场景： 讨论设计约束或早期决定带来的限制

📌 出处： The Guardian, 2022

“The memory map isn’t set in stone — if you run out of RAM, you can reorganize sections or enable the external SRAM.”

4. “the devil is in the details”

含义： 细节决定成败；看似简单的事情往往有复杂的细节
场景： 讨论链接脚本、启动代码等细节复杂的技术话题

📌 出处： 古谚语，广泛引用

“Linking looks trivial until you trace a boot failure — the devil is in the details of symbol names, alignment, and KEEP directives.”

5. “put it into perspective”

含义： 正确看待；从更广的视角理解
场景： 面试中解释复杂概念时引入宏观视角

📌 出处： 常见口语表达

“To put it into perspective: every embedded program you’ve ever run went through a linker script — you just never had to write one before.”
