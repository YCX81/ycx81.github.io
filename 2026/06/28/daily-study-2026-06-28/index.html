🔧 嵌入式知识：LoRa广域网通信

原理

LoRa（Long Range）是 Semtech 专有的线性调频扩频（CSS, Chirp Spread Spectrum）物理层，工作于 Sub-GHz ISM 频段（中国 470MHz、欧洲 868MHz、美国 915MHz）。核心特性：

  • 超长距离：城区 2~5km，郊区 10km+，视距 15km 以上。
  • 超低功耗：发射 100mW（20dBm）下峰值 ~120mA，但配合深度睡眠全年可靠电池跑。
  • 超低速率：几百 bps 到几十 kbps，不是给视频用的。

关键物理参数：

  • 扩频因子 SF（Spreading Factor）：SF7~SF12。每增加 1，空中时间翻倍、灵敏度提升 ~2.5dB、距离更远。
  • 带宽 BW：125 / 250 / 500 kHz。BW 越大速率越高、抗多径越强，但占用频谱。
  • 编码率 CR：4/5、4/6、4/7、4/8，纠错强度换速率。
  • 数据速率：典型 SF7/BW125 ≈ 5.5kbps，SF12/BW125 ≈ 250bps。

LoRaWAN 是建立在 LoRa PHY 上的 MAC 层，ClassA/B/C 三种工作模式：

  • Class A：上行完后开两个短接收窗口（RX1 / RX2），功耗最低，适合电池设备。
  • Class B：节点在 Beacon 同步下周期性开接收窗，低功耗 + 下行可控。
  • Class C：一直开接收（除了上行时），下行任意时刻，功耗高，但响应快。

典型 LoRaWAN 架构：End Device → Gateway（集中器，多通道同时监听）→ Network Server（调度 / 去重 / ADR）→ Application Server（业务）。

关键代码（SX1276 点对点传输 + 深度睡眠）

易混淆点

  1. LoRa vs LoRaWAN：LoRa 只是物理层，LoRaWAN 是基于它的通信协议栈（入网、加密、ADR）。点对点小网络完全可以只用 LoRa PHY + 自研协议。
  2. 速率与距离不能兼得：SF12 最远但空中时间长，占空比法规限制更严（欧洲 1% duty-cycle 可能就发不了大包）。
  3. 频段合规：中国 470~510MHz 民用，不允许 868/915；欧美则相反；硬件可能相同但频段软件定死。
  4. 集中器工作原理：不是单路扫描，而是 8 通道 LoRa 同时解调（Semtech SX1301 / SX1302），以及 1 路 GFSK。
  5. 电池寿命计算：要把 Tx 电流 × Tx 时间 + Rx 电流 × Rx 窗口 + 睡眠电流 × 睡眠时间，全部加起来再除以容量；只算 Tx 会高估一年以上。
  6. 地面反射多径：LoRa 抗多径强，但低矮天线仍会出现 fading；现场多用 90 度偏振方向实验。

面试常见问法

Q1：LoRa 能达到 10km，为什么 WiFi 才 100m？
A：① Sub-GHz 频段波长长、绕射强，穿墙优于 2.4GHz；② 扩频把信号铺在宽带中，接收灵敏度到 -140dBm（比 Wifi 好 40dB）；③ 数据率极低（几 kbps），信号带宽窄，处理增益高。代价就是带宽小，无法传图片视频。

Q2：为什么 LoRaWAN Class A 功耗最低？
A：节点发送一次上行后，只在 RX1（频率与上行一致）和 RX2（固定下行信道）两个短时间窗打开接收，其他时间全部深度睡眠。下行只能在节点上行之后，Gateway 排队调度；这对遥控类应用不合适，但对传感器非常省电。

Q3：如何提升 LoRa 节点的工作稳定性？
A：① 天线要加匹配网络并做 VSWR 测量；② 时钟用 TCXO（Semtech 要求 ±15ppm），普通 XO 温度漂移会丢同步；③ 加 ADR（Adaptive Data Rate）让服务器动态调 SF；④ 注意电池内阻，发射瞬间电压下跌 MCU 复位；⑤ 统计 PER（包错误率）做远程诊断。

Q4：LoRaWAN 安全吗？
A：协议层两级 AES-128 密钥：NwkSKey（网络完整性 + MIC）、AppSKey（应用数据加密）。下行有 frame counter 防重放。但需要关心 OTAA/ABP 入网方式：OTAA 每次入网动态协商密钥，较 ABP 更安全；密钥分发全靠私人管理，厂商平台的密钥保管是系统短板。
