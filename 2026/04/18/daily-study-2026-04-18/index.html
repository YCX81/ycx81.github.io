🔧 嵌入式知识：Git版本控制与协作

原理

Git分布式版本控制，每个开发者有完整仓库。支持分支、合并、冲突解决。是现代软件开发必备工具。

关键代码

易混淆点

  1. Merge vs Rebase: Merge保留历史分支,Rebase线性但改写历史
  2. Fast-forward: 直接祖先关系可快进
  3. Detached HEAD: 检出旧提交进入分离头指针状态

面试常见问法

Q1: Merge和Rebase区别？
A: Merge创建Merge Commit,保留分支历史。Rebase改写历史变线性,好看但改写了历史。

Q2: 如何撤销已推送的提交？
A: git revert创建新提交撤销,或git reset + git push –force重写(危险,会影响他人)。
