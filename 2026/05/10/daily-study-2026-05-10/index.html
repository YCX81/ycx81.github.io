🔧 嵌入式知识：无线通信与低功耗蓝牙

原理

嵌入式常见近距离无线：蓝牙（BR/EDR）、BLE、WiFi、Zigbee、Thread、Matter、Sub-GHz（LoRa/Zwave）、UWB。其中BLE（Bluetooth Low Energy，蓝牙 4.0+） 因为低功耗、协议简单、手机原生支持，在 IoT 产品里渗透率最高。

BLE 协议栈自底向上：

  • 物理层（PHY）：2.4GHz ISM，40 个信道（37 个数据 + 3 个广播），GFSK 调制，1Mbps（BLE4）或 2Mbps/LE Coded PHY（BLE5）。
  • 链路层（LL）：角色（Advertiser/Scanner/Initiator/Central/Peripheral）、连接参数（Interval、Latency、Timeout）、加密（AES-CCM）。
  • HCI：Host-Controller 分界线。
  • 主机层：L2CAP / ATT / GATT / SMP / GAP：ATT 提供“属性”键值读写，GATT 把属性组织成 Service + Characteristic 层级，SMP 做配对/加密，GAP 管角色和发现。
  • 应用层 profile：HID、Heart Rate、Battery、Custom 等预定义或自定义 profile。

典型工作流程：Peripheral 周期广播 → Central 扫到发起连接 → 建连接后 GATT 互相 discover Service/Characteristic → Central 订阅 Notify 或发 Write → 数据双向流动。广播参数（Interval 20ms10s）和连接参数（Interval 7.5ms4s）直接决定延迟和功耗。

低功耗关键点：大部分时间睡眠，只在广播/连接事件点亮无线几百 μs。合理设计下 CR2032 纽扣电池可跑 1~3 年。常见硬件：nRF52、ESP32（双模 WiFi+BLE）、DA14585、CYW20819。

易混淆点

  1. 广播 ≠ 连接：广播可被任意设备扫到；连接是 1 对 1 的会话。大量传感场景用 Broadcaster + Observer 模式不建立连接，更省电。
  2. Interval vs Latency：连接 Interval 是“多久握一次手”，Slave Latency 是“Slave 可以跳过多少次不响应”；组合调整可在延迟和功耗间折中。
  3. MTU：默认 23 字节（ATT_MTU）；BLE5 可协商到 247 字节以上，上层高速应用都需要把 MTU 协商高。
  4. Pairing / Bonding：Pairing 是本次会话加密，Bonding 把密钥持久化到下次连接复用；Just Works / Passkey / OOB / Numeric Comparison 四种方式安全级别不同。
  5. TX Power 越大越好？：大功率发射虽远，但接收灵敏度不变，近距离干扰自己且耗电，距离受限多为障碍物非功率。
  6. 操作系统 BLE 栈权限：iOS 要 App 在后台保活比 Android 更复杂；必要时用 Service UUID 白名单或 iBeacon/UUID Advertising 模式。

面试常见问法

Q1：BLE 和经典蓝牙 BR/EDR 区别？
A：两者在 2.4GHz 上完全不同的物理层和协议。BR/EDR 面向音频/文件传输，持续连接、峰值几 Mbps；BLE 面向低频率小数据，连接事件间歇运行，功耗比 BR/EDR 低一个数量级。双模芯片同时支持，但用于不同场景。

Q2：如何估算 BLE 设备功耗？
A：主要看广播/连接事件数量 × 每次事件平均电流 × 事件时长 + 空闲睡眠电流。典型 1s 广播 + 无连接，Peripheral 平均电流 10~30μA；连接 1s Interval、Latency=0 下 50μA 左右。再乘以时间。

Q3：BLE 安全怎么做？
A：配对选 LE Secure Connections（基于 ECDH）防中间人；开启 Bonding 持久化密钥；对关键 Characteristic 设置 need_encryption、need_authenticated_write；加上应用层 AES/Signed Write 再加一道锁。厂商私钥烧 OTP 而非写 Flash。

Q4：BLE 数据吞吐怎么提升到 1Mbps+？
A：① 协商 ATT_MTU 到 247B；② 使用 Data Length Extension（DLE）把 PDU 拉到 251B；③ 启用 2M PHY（BLE5）；④ 收紧连接 Interval 到 7.5ms；⑤ 启用多次 Notification Per Event。综合起来可到 1.3Mbps 应用层吞吐。
