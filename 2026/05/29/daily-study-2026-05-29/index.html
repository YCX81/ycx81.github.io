🔧 嵌入式知识：Modbus协议实现

原理

Modbus 是 Modicon 公司 1979 年为可编程控制器（PLC）开发的主从式串行协议，现已成为工业自动化最普及的开放协议——温度变送器、变频器、HMI、SCADA、IoT 网关几乎都支持 Modbus。核心设计简单直观：Master 发 Request、Slave 回 Response、每次仅一个主机在总线活跃，容易实现且兼容性极佳。

协议分三种变体：Modbus RTU（Remote Terminal Unit，串口 RS-485/RS-232 上二进制编码）、Modbus ASCII（串口上十六进制可打印字符编码，开销大但人类可读）、Modbus TCP（以太网，去掉 CRC 加 MBAP 头，默认端口 502）。RTU 最常见，工业现场 RS-485 总线布线，一主多从（最多 247 个从站）。

PDU（Protocol Data Unit，应用层）由 Function Code（1 字节）+ Data 组成，常用功能码：0x01 Read Coils（读线圈，位量输入）、0x02 Read Discrete Inputs、0x03 Read Holding Registers（读保持寄存器，16 位量）、0x04 Read Input Registers、0x05 Write Single Coil、0x06 Write Single Register、0x0F Write Multiple Coils、0x10 Write Multiple Registers。异常响应功能码最高位置 1（如 0x83 响应异常）。

RTU 帧格式：Slave ID（1B）+ PDU（N B）+ CRC16（2B）。帧间需要静默时间（3.5 字符时间，9600 bps 下约 4ms）作为帧分隔，这是 RTU 特有的”时间分隔”机制而非起止符。实现要点：接收时监控字符间隔 > 1.5 字符时间判断帧边界，> 3.5 字符时间判断帧结束。

Modbus TCP 帧格式：MBAP Header（Transaction ID 2B + Protocol ID 2B=0 + Length 2B + Unit ID 1B）+ PDU（功能码 + 数据）。没有 CRC（TCP 已保证可靠），Transaction ID 让主站可并发多请求。地址映射与 RTU 相同，方便网关转换。

易混淆点

  1. RTU vs ASCII：RTU 二进制紧凑，ASCII 用 “:” 起始 “\r\n” 结束的可打印字符
  2. 寄存器地址 vs PDU 地址：40001 ~ 49999 是 Holding Register 传统编号，实际 PDU 地址 0x0000 ~ 0x2710
  3. 字节序：Modbus 寄存器内高字节在前（Big-Endian），多寄存器浮点有 4 种字节组合
  4. 广播 ID 0：广播地址，从站必须响应但不回复

面试常见问法

Q1：Modbus 为什么这么流行？
A：简单（无复杂握手/状态机）、开放（免费规范）、广泛支持（几乎每个工业设备都实现）、低带宽（RS-485 1.2kbps 到 10Mbps 都能跑）。

Q2：Modbus 实时性如何？
A：主从轮询本质不是实时协议，周期时间随从站数量和帧长线性增长。工业场景 100ms 周期很典型，对毫秒级控制用 CAN/EtherCAT。

Q3：Modbus 没有安全机制，生产上怎么办？
A：VPN/专网隔离、加 TLS 层（Modbus/TCP over TLS RFC 草案）、网关层做访问控制。传统现场总线依赖物理隔离防护。
