🔧 嵌入式知识：浮点数精度与定点数运算

原理

嵌入式里“算数精度”话题主要绕在两个选择上：浮点（IEEE 754 float/double） 还是 定点（Q-format）。两者各自优缺点决定了不同场景的用法。

IEEE 754 float（32bit）结构：1bit 符号 + 8bit 阶码（偏移 127）+ 23bit 尾数。表示范围 ≈ ±3.4e38，有效位约 7 位十进制；double 64bit 有效位 15~16 位。优点：范围宽、写起来自然；缺点：

  • 硬件依赖：Cortex-M4F / M7 / M33F 有 FPU，单指令完成；Cortex-M0/M3 无 FPU，软件模拟一次乘法几十到几百周期。
  • 非均匀精度：1.0 + 1e-8 会被吞掉，0.1 + 0.2 ≠ 0.3。
  • **比较慎用 ==**：浮点比较要用差值 < epsilon。

定点数（Fixed-Point / Q-format）：把整数按约定的“小数点位置”解释。Q16.16 表示 32bit 中高 16 位整数、低 16 位小数；Q15 表示 16bit 有符号，小数 15 位（[-1, 1)）。优点：

  • 用整数指令做加减乘除，无 FPU 也能高效算。
  • 确定性精度，便于数字信号处理（FIR、IIR、FFT）硬件实现。
  • 乘法结果位宽翻倍，要右移回到原 Q 格式。

转换与运算：

工程上还有 定点浮点混合：对控制环的核心部分用定点（稳定、可重复），对配置、日志、校准用浮点（方便）；FIR 滤波器系数提前转 Q 格式，避免运行时频繁转换。

易混淆点

  1. float 开 FPU 就“不慢”了吗？：开 FPU 后基本四则运算单周期，但 sinf/expf 这类数学函数仍是软件库（几百周期），热点循环里不宜乱用。
  2. 单精度够不够？：7 位十进制对测量数据常足够，但控制环累积误差可能显著，必要时升 double 或定点。
  3. printf("%f") 在 MCU 默认不支持：newlib-nano 去掉了浮点支持，链接时要加 -u _printf_float 并预留栈空间，否则打印出现乱码或 nan。
  4. 定点饱和：加法/乘法溢出时要饱和截断（ARM DSP 有 SAT 指令），否则一个 tick 突变就把控制量打飞。
  5. Q 格式一不一致：混 Q15 和 Q16 运算却没移位对齐，结果错得很隐蔽；团队里必须统一格式并注释每个变量的 Q。
  6. 浮点中断保存开销：FPU 打开后进入 ISR 要压额外 18 个寄存器，中断延迟增加；不用浮点的 ISR 可关 Lazy Stacking。

面试常见问法

Q1：Cortex-M3（无 FPU）要做 DSP 怎么办？
A：一律定点。用 CMSIS-DSP 库的 q15/q31 版本函数，它们内部用 DSP 扩展指令（SMLAD、SMULL）在 Cortex-M4/M7 上高速，M3 也能靠 MLA 跑，但周期较长。设计时提前估算需要的精度位数，选 Q15 或 Q31 做乘累加。

Q2：控制系统 PID 用浮点还是定点？
A：汽车、工业控制为了可重复性和跨平台一致性多用定点；消费电子、原型开发用浮点写得快。一旦量产要做形式化验证（AUTOSAR / IEC 61508），定点更容易进行数值边界分析。

Q3：浮点精度丢失有哪些典型症状？
A：① 大数加小数：1e8 + 0.1 = 1e8；② 相近减法灾难：两个接近数相减结果只剩噪声位；③ 误差累积：FFT 大量乘加误差叠加；④ 比较不相等：0.1+0.2 ≠ 0.3；⑤ NaN 传染：一次除零全部链条变 NaN。

Q4：定点数饱和怎么处理？
A：三种办法：① 用 32bit 累加器扛溢出，最后再截回 16bit；② 检测超界饱和到最大/最小；③ ARM 的 __QADD16/__SSAT 指令一步饱和。DSP 滤波器里一般直接用 CMSIS-DSP 的 Q 版本，它内部已经处理饱和。
